# py-ngrokd

基于Python 3.5+，asyncio的Ngrok服务端，正在优化中，目前只支持HTTP/HTTPS代理

(原本Ngrok的TCP代理感觉不合理，原来的实现是一个客户端（需要代理的机器）会占用服务器的一个端口，个人觉得应该做成像HTTP/HTTPS那样，只占用1个端口根据URL（或者类似的地址）进行对应的转发，但这样需要客户端和服务端都修改，所以暂时不实现TCP的代理)
